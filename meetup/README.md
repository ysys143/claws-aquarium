# OpenClaw 서울 빌더 밋업 2026-03-15 전체 요약

## 1. 행사 개요

**일시**: 2026년 3월 15일
**개최**: Instruct.KR (Discord 기반 AI 개발자 커뮤니티)
**규모**: 공간 수용 50~60명 / RSVP 950명 이상 (역대 최다 등록)
**의미**: OpenClaw 서울 최초 개최 + ClawCon 서울 공식 유치 확정

OpenClaw 서울 빌더 밋업은 AI 에이전트 개발자와 실무자들을 위한 한국 최초의 대규모 밋업이다. 공간 대비 16~19배의 RSVP 수요는 한국의 AI 에이전트 개발 열풍을 반영한다. 특히 OpenAI 창업자 피터가 주도하는 글로벌 ClawCon 순회 행사의 서울 유치가 확정되며, 서울이 아시아 AI 개발 중심지로 부상하고 있음을 보여준다.

## 3. 주제별 클러스터링

### 인프라 & 아키텍처

**다중디바이스 통합 제어**
- 최재훈: OCI 무료 인스턴스를 중앙 게이트웨이로 삼아 Mac/라즈베리파이/Windows를 OpenClaw로 통합 제어. Tailscale VPN으로 네트워크 단순화.
- 이상현: 클라우드 네이티브 서버리스 구조(AWS Lambda + TypeScript + MQTT)로 로컬 머신 노출 없이 안전한 에이전트 운영.

**하드웨어 & 로컬 AI**
- 진주성: M3 Ultra 512GB 1대로 QWen 35B/122B/128GB 모델을 동시 운영. 3티어 전략(로컬 소형→중형→클라우드 소타)으로 비용 최적화.
- 민용기: 채 템플릿 튜닝으로 로컬 모델 성능 7~8% 향상. oh-my-openclaw 프로젝트로 한국 개발자 워크플로우 최적화.

### 에이전트 운영 & 오케스트레이션

**팀처럼 에이전트 관리**
- 허예찬: "AI 팀 운영"의 핵심은 명확한 행동 규칙(agents.md), 파일 시스템 기반 메모리, 그리고 슈퍼가제 단일화. OMC/OMX의 80% 이상 PR을 에이전트가 생성.
- 윤주운: FLOCK(홈랩 분산 에이전트) + Arbiter(메타 에이전트)로 자연어 명령→워크플로 자동 생성. Mattermost 호스팅으로 프라이버시 확보.
- 정세민: Slack 연동 + 휴먼-인-더-루프 구조로 4단계 워크플로(탐색→리스트→스펙→구현) 자동화. 하이브리드 검색(BM25+벡터) + 온톨로지로 장기 기억 보완.

**이동 중 개발**
- 정우석: Claude Code와 OpenClaw의 역할 분담(기획·검수 vs 복잡한 제작)으로 비개발자도 스킬 개발 가능. 4단계 프로세스(요청→실행→고정→트리거) 정립.

### 온체인 에이전트 경제

**블록체인 기반 거래**
- Logan Kang: OpenClaw(에이전트 런타임) + Virtuals ACP(마켓플레이스) + Base(USDC 결제)로 에이전트 간 자율 거래. X402(HTTP 402 기반 결제 미들웨어) + 에스크로 구조.
- 김서준: AI 에이전트 경제의 4가지 기둥 = MCP(명함) + A2A 통신 + X402 결제 + ERC-8004 신원 인증. 한국 생태계의 API 폐쇄 문제 지적.
- Zoe Chen: Unibase로 탈중앙화 메모리 + AIP(에이전트 상호운용) + Beat Agent(토큰 런칭·자율 트레이딩) 구현. 에이전트가 기억 유지하며 블록체인 거래 독립 수행.

### 세컨드 브레인 & 개인 생산성

**AI 비서 시스템**
- 김우현: 17만원 중고 미니 PC + Claude Code + Obsidian + Discord 삼각 구조. CODE 프레임워크(수집→정리→증류→표현) + AI 자가 발전.
- 김동규: nanoclaw(4,000줄) 포크로 커스텀 에이전트 시스템. 도비(오케스트레이터), 홍명보(마케터), 투두몬(태스크) 등 5개 에이전트 팀 구성. 공유 지식 저장소(Obsidian).

### 기술 심화

**툴 콜링 & 해시라인 편집**

## 4. 밋업 전체를 관통하는 인사이트

**1. 에이전트는 도구에서 팀으로 진화 중**
- 발표들을 보면 단순 챗봇을 넘어 명확한 역할·메모리·행동 규칙을 갖춘 팀원으로서 에이전트를 운영하는 추세. 허예찬의 "AI 팀 운영"이 이를 잘 보여줌.

**2. 로컬 vs 클라우드의 최적 균형점이 형성 중**
- 진주성의 3티어 전략(로컬 소형→중형→클라우드 소타)처럼, 비용과 성능의 트레이드오프를 명확히 인식하고 설계하는 패턴이 확산 중.

**3. 메모리 아키텍처가 경쟁력의 핵심**
- 파일 시스템(허예찬), 하이브리드 검색(정세민), 탈중앙화 메모리(Zoe Chen)처럼 에이전트의 장기 기억을 어떻게 설계하느냐가 실무 효율을 크게 좌우.

**4. 휴먼-인-더-루프 구조의 재발견**
- 완전 자동화보다 사람의 승인·통제 루프를 단계별로 유지하는 구조(정세민, 윤주운)가 실제 운영에서 더 안정적이고 신뢰성 있음.

**5. 온체인 에이전트 경제는 기술이 아닌 신뢰·신원 문제**
- Logan Kang, 김서준, Zoe Chen의 발표에서 ERC-8004, X402, 에스크로 등 기술보다 "에이전트가 신뢰할 수 있는 신원 인증 표준"이 진정한 병목임을 지적.

**6. 개발자 경험의 민주화**
- 정우석, 김우현처럼 비개발자도 Claude Code의 4단계 프로세스나 Claude Hub의 기존 에이전트 조합만으로 실용적인 자동화 구현 가능해짐.

**7. 한국 생태계의 API 폐쇄가 큰 한계**
- 김서준 지적: 한국 대기업·빅테크가 API를 공개하지 않으면서 에이전트 생태계 발전이 더딤. 글로벌 오픈소스 커뮤니티 중심으로 움직여야 한다는 조언.

## 7. OpenClaw에 주목해야 하는 이유

**기술적 관점: 개인이 소유하는 에이전트 런타임**

OpenClaw는 단순한 도구가 아니라 AI 에이전트를 직접 소유하고 운영할 수 있게 해주는 런타임 플랫폼이다. 이번 밋업의 발표들이 실증하듯, 로컬 머신에서 OCI 무료 인스턴스, AWS 서버리스(Lambda), 클라우드까지 다양한 인프라에 유연하게 배포 가능하다. 더 나아가 MCP(Model Context Protocol), A2A(Agent-to-Agent), X402(결제 표준) 같은 새로운 에이전트 표준의 구현 계층으로 기능하며, 스킬 시스템으로 기능을 모듈화해 비개발자도 확장할 수 있다.

**커뮤니티 관점: 빌더 생태계의 성장**

OpenClaw는 이미 글로벌 ClawCon(샌프란시스코 2,200명 참석)을 중심으로 확산 중이며, 서울 ClawCon 공식 유치로 아시아 거점을 확보했다. 오픈소스 기반으로 nanoclaw, oh-my-openclaw, OMC/OMX 같은 파생 프로젝트들이 활성화되고 있으며, 단순 사용자가 아닌 직접 구축하는 빌더(Builder) 커뮤니티가 형성되는 중이다.

**실용성 관점: 이미 검증된 활용 사례**

이번 밋업에서 발표된 17만원 중고 PC의 세컨드 브레인, 서버리스 에이전트, 연구 자동화, 다중 디바이스 제어 등은 모두 현재진행형의 사례다. 특히 정우석, 김우현 같은 비개발자들도 실제 업무에 활용 중이며, 기존 SaaS 대비 데이터 주권 확보와 비용 최적화가 가능하다는 점이 강점이다.

**미래 관점: 에이전트 경제의 주도권**

Logan Kang의 Composable Agent Economy와 김서준의 온체인 에이전트 경제 분석에서 드러나듯, "누가 에이전트 런타임을 장악하느냐"는 다음 AI 플랫폼 전쟁의 핵심이다. OpenClaw는 ERC-8004(신원 인증), X402(결제), 탈중앙화 메모리 같은 신뢰 아키텍처와 결합하며, 온체인 생태계와 연결 가능한 개방형 구조를 갖추고 있다.

**한 줄 결론**: OpenClaw는 개인 소유의 에이전트 런타임에서 출발해 빌더 생태계, 온체인 경제까지 아우르는 "에이전트 경제의 기반 인프라"로 진화 중이다.

## 요약

OpenClaw 서울 빌더 밋업 2026-03-15는 한국의 AI 에이전트 개발 생태계가 성숙 단계에 접어들었음을 보여주는 행사다. 단순 자동화 도구에서 벗어나 명확한 신원, 장기 기억, 상호운용성을 갖춘 "팀으로서의 에이전트" 운영이 주류가 되고 있으며, 온체인 경제를 지탱할 신뢰 아키텍처(ERC-8004 등)도 동시에 형성 중이다. 로컬 LLM의 실용화, 휴먼-인-더-루프 패턴의 정착, 그리고 API 폐쇄 문제 인식이 향후 과제이며, ClawCon 서울 유치로 아시아 AI 개발 중심지로서의 한국의 위상이 한층 높아질 것으로 기대된다.

## 8. OpenClaw는 Claude Code + Remote Control과 무엇이 다른가

### Claude Code + SSH/Tailscale 조합의 한계
- 원격 머신에 접속해서 Claude Code로 코드 작성 가능
- 사람이 명령 입력 → Claude가 응답 → 사람이 다음 명령
- **핵심 한계: 사람이 매번 트리거해야 함. 세션 종료하면 끝.**

### 결정적 차이

| 구분 | Claude Code + Remote | OpenClaw |
|------|----------------------|----------|
| 실행 방식 | 사람이 매번 시작 | 24시간 상시 대기, 이벤트로 자동 실행 |
| 트리거 | 사람의 입력 | 크론잡, 웹훅, 메시지, 파일 변경 등 |
| 메모리 | 세션 내에서만 | 영속적 메모리 시스템 |
| 멀티디바이스 | SSH 하나씩 | 게이트웨이-노드 구조로 여러 기기 동시 오케스트레이션 |
| 스킬 시스템 | 없음 | 설치·조합 가능한 모듈 단위 기능 확장 |
| 메신저 통합 | 없음 | Telegram·Discord·Slack 네이티브 연동 |
| 멀티 에이전트 | 없음 | 여러 에이전트가 채널로 협업하는 구조 내장 |

### 밋업 사례로 본 실증

- **진주성**: SNS를 4시간마다 자동 크롤링 → 사람이 아무것도 안 해도 텔레그램으로 리포트 수신
- **정우석**: 관심 채널을 자동 모니터링 → 신규 콘텐츠 감지 시 자동 요약·발행
- **김동규**: GitHub 이슈 발생 → 자동으로 Codex 호출 → PR 생성 (사람 개입 0)
- **허예찬**: OMC/OMX PR의 80%를 에이전트가 스스로 생성

### 한 줄 요약

> **Claude Code는 내가 쓰는 도구, OpenClaw는 나 대신 일하는 직원**

Claude Code + Remote는 사람이 운전대를 잡고 있어야 한다. OpenClaw는 목적지와 규칙만 주면 알아서 달리는 구조다. 이 차이가 "자동화"와 "에이전트" 사이의 간극이다.

## 9. OpenClaw는 유행이 아니다: AI 비서 OS로의 진화와 사업화

이번 밋업의 발표들이 보여주는 더 큰 맥락은 OpenClaw가 "개발자 장난감"을 넘어 실제 상용 제품의 기반이 되고 있다는 점이다. 밋업 직후 국내 팀이 오픈 베타를 시작한 **Crew**(crew.you)는 이를 잘 보여주는 사례다. 카카오톡, 텔레그램, Slack 등 7개 메신저 연동, Gmail/Google Calendar/M365 통합, 50여 가지 빌트인 도구, 스킬 마켓플레이스 — 밋업에서 개발자들이 직접 구현해 발표한 것들이 소비자 제품으로 패키징된 형태다.

이 흐름이 의미하는 바는 명확하다. OpenClaw 계열 스택은 이미 **"Agent OS"** — 개인의 디지털 업무를 위임받아 24시간 처리하는 운영체제 — 로 성숙하고 있다. 스마트폰 OS 위에 앱이 쌓였듯, Agent OS 위에 스킬/에이전트가 쌓이는 구조다. 사업화 관점에서 이 레이어를 선점하는 것은 단순한 SaaS 출시가 아니라 차세대 개인 생산성 플랫폼의 주도권을 갖는 것과 같다.

개발자가 먼저 만들고, 커뮤니티가 검증하고, 제품이 따라오는 이 패턴 — OpenClaw 생태계는 이미 그 사이클을 돌리고 있다.
