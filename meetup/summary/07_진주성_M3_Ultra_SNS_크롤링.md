# 발표 요약: 진주성 - M3 Ultra 512GB 듀얼 활용 OpenClaw 무제한 SNS 크롤링 후기

## 개요
- 삼성전자 출신 AI 개발자 진주성이 M3 Ultra 512GB 맥미니를 구매하여 로컬 LLM을 운용한 경험을 공유한다. 24시간 SNS 크롤링 기반 금융 자산 매력도 변화 감지 시스템을 구축하면서 로컬 LLM이 클라우드 API 대비 비용 측면에서 얼마나 유리한지를 실제 사례를 통해 설명한다. 3티어 모델 전략(로컬 소형 → 로컬 중형 → 클라우드 소타)을 통해 효율적인 AI 워크플로우를 설계하는 방법을 제안한다.

---

## 주요 주제/섹션

### 1. 발표자 소개 및 로컬 LLM 도입 배경 [04:35:15]
- 삼성전자에서 서비스 AI(챗봇 등) 개발 후 2024년 독립, 창업 및 MBA 수료 중
- 홈 AI 붐으로 인해 개인 서버 운용 시 LLM 호출 비용이 폭증할 것을 예측
- Claude Code, Agentic 워크플로우가 클라우드 모델에 과도하게 의존하는 구조에 문제 인식
- 비용 절감 및 온프레미스 사업 가능성을 위해 로컬 LLM 전환 결정

### 2. M3 Ultra 512GB 구매 결정 과정 [04:36:34]
- 유튜브에서 M3 Ultra 4대를 클러스터링한 영상을 보고 구매 결심
- M3 Ultra 1대(512GB) 약 1,600만 원, NVIDIA H100 기준으로는 동급 메모리를 위해 7대 필요 → 투자비·전기비·서버실 비용 비교 우위
- 맥미니의 작은 폼팩터와 낮은 전력 소비를 장점으로 평가
- Llama 2.5 운용 목적으로 2대 구매 결정 (2번째 기기 배송 중, 현재 512GB 옵션 품절)
- 듀얼 클러스터는 썬더볼트로 풀메시 직접 연결 필요, 1TB+ 통합 메모리 구성 가능

### 3. M5 출시 이후 M3 Ultra 포지셔닝 재검토 [04:39:09]
- M5 발표로 인해 M3 Ultra의 상대적 가치 하락 인식
- M5 Max가 프리필 속도(6~10초 → 3~5초)와 토큰 생성 속도(80 → 100 토큰/초) 모두 개선
- M5 Ultra 출시(여름 이후 예상) 시 M5 Ultra 512GB로 업그레이드 계획
- 결론: 듀얼 클러스터는 현시점에서 불필요, 단일 울트라 1대로 충분

### 4. 현재 서빙 중인 모델 구성 [04:41:10]
- QWen 3.5 35B: 약 35GB (MLX 4비트 양자화 기준)
- QWen 3.5 122B: 약 70GB
- MiniMax: 약 128GB
- 총합 256GB 미만으로 512GB 내에서 여유롭게 운용
- LM Studio를 통해 맥북에어에서 원격 모델 접근 가능
- 벤치마크상 해당 모델들은 GPT-4 mini급 이상 성능으로 평가

### 5. 3티어 모델 전략 [04:43:33]
- **티어 1 (게이트키핑)**: QWen 3.5 35B — 빠른 속도, 낮은 메모리(액티브 3B), 대량 필터링
- **티어 2 (복잡한 추론)**: QWen 3.5 122B — 중간 복잡도 분석
- **티어 3 (최종 의사결정)**: GPT-4.1 (ChatGPT Pro 월 30만 원 구독 활용)
- API 비용을 소타 모델에만 집중 투입하는 경제적 구조 설계

### 6. SNS 크롤링 기반 금융 자산 매력도 감지 프로젝트 [04:44:34]
- MBA 현장 프로젝트 주제: "자산 훼손 감지" (주식·채권·원자재 수급 변화 모니터링)
- 크롤링 도구: Apify X Scraper (월 30~50달러), X(트위터) 키워드 크롤링
- 24시간 연속 크롤링으로 자산 매력도 변화 신호 감지
- 채권 전문가 에이전트, 골드 전문가 에이전트 등 도메인별 서브 에이전트 구성
- 오케스트레이터가 각 에이전트 결과를 종합하여 최종 신호 판단
- 텔레그램 봇으로 신호등(불리시/베어리시) 및 국가별 현황 리포트 출력 구현 완료

### 7. Q&A: 로컬 LLM 선택 이유 [04:48:08]
- 질문: 금융 데이터는 보안이 덜 중요한데 왜 로컬을 선택했나?
- 답변: 24시간 전 세계 정보를 크롤링하여 계속 처리하면 클라우드 LLM 비용이 감당 불가
- 로컬 모델로 1차 가능성 체크를 하고, 의미 있는 데이터만 클라우드 소타 모델로 전달

## 핵심 포인트 및 세부 내용
- M3 Ultra 512GB는 NVIDIA H100 서버 7대 분량의 메모리를 단일 저전력 기기로 대체 가능
- MLX 4비트 양자화 적용 시 QWen 122B도 약 70GB로 운용 가능
- QWen 3.5 35B는 액티브 파라미터 3B 수준으로 MoE 구조 덕분에 속도·메모리 효율 우수
- 128GB 모델만 있어도 로컬 LLM 운용에 큰 제약 없음 (발표자 권장 입문 사양)
- ChatGPT Pro(월 30만 원) 구독 시 o1, GPT-4.1 등 최상위 모델을 무제한에 가깝게 사용 가능
- API 비용을 "생산성 투자"로 바라보는 시각 전환 권장 (비용 아끼기보다 생산성 극대화)
- X 크롤링은 Apify X Scraper로 비교적 저렴하게 구현 가능 (월 30~50달러)
- 금융 API와 결합하면 더욱 정교한 자산 신호 탐지 모델 구축 가능

## 결론 및 권장사항
- 24시간 대규모 크롤링·처리 워크플로우에서 로컬 LLM은 클라우드 대비 압도적인 비용 우위를 가지며, 단일 M3 Ultra 1대만으로도 실용적인 멀티 에이전트 시스템 구축이 가능하다.
- 소타급 추론이 필요한 최종 의사결정 단계에만 클라우드 API를 활용하는 3티어 전략이 비용과 성능의 최적 균형점이며, M5 시리즈 출시 이후에는 단일 울트라 기기로도 충분한 성능을 기대할 수 있다.
