# 요약: Remote OpenClaw로 라즈베리파이5와 맥 스튜디오 컨트롤하기

발표자: 최재훈 | 발표 시간: 13:30~13:50 | 녹취 구간: [02:57:17]~[03:18:01]

---

## 개요

온디바이스 AI와 IoT 분야를 다루는 발표자가 OpenClaw를 활용해 맥 스튜디오, 라즈베리파이5, 윈도우 PC(게임기) 등 여러 기기를 단일 리모트 게이트웨이로 동시 제어하는 구성을 소개한다. 오라클 클라우드(OCI) 무료 인스턴스를 중앙 게이트웨이로 활용하는 방법과, OpenClaw의 로컬/리모트 게이트웨이 및 노드 구조를 튜토리얼 형식으로 설명한다.

---

## 주요 주제/섹션

### 1. 오라클 클라우드(OCI) 무료 인스턴스 활용 [02:57:56]
- 4 CPU, 24GB RAM, 200GB 스토리지를 아웃바운드 10TB 이내에서 평생 무료로 제공
- 인스턴스 생성 시 CPU 수를 늘리는 옵션이 UI에 숨겨져 있어 주의 필요 (화살표 클릭으로 확장)
- 인스턴스 생성이 즉시 안 될 경우 두 가지 방법: 스크립트로 반복 시도 or 페이즈(Pay-as-you-go) 업그레이드
  - 페이즈 방식은 약 30분 내외로 훨씬 빠르게 할당 가능
  - 비용 경고 알림(1달러~10달러)을 설정해 실수 방지 권장
- Tailscale로 전체 디바이스를 VPN 망으로 묶으면 네트워크 설정 단순화 가능

### 2. OpenClaw 아키텍처: 로컬 vs. 리모트 게이트웨이 [03:02:03]
- **로컬 게이트웨이**: 에이전트 런타임과 같은 머신에서 동작, `exec`로 직접 로컬 리소스 접근 가능
- **리모트 게이트웨이**: 클라우드(OCI 등)에 위치, 별도 원격 디바이스의 리소스를 직접 `exec`로 접근 불가
  - 리모트 게이트웨이에서 다른 머신을 제어하려면 반드시 **OpenClaw 노드(Node)** 를 통해야 함
  - 구조: 리모트 게이트웨이(에이전트 런) → 각 디바이스에서 실행 중인 노드 → 실제 OS 기능 수행

### 3. OpenClaw 노드(Node) 개념과 역할 [03:05:02]
- 노드는 자신의 기능(Capabilities)을 게이트웨이에 Advertisement(광고/등록)
- 컴패니언 앱(Companion App)에서 등록된 노드 및 각 노드의 Capabilities 확인 가능
- Capabilities는 OS마다 다름 (macOS, Linux/라즈베리파이, Windows 각각 상이)
  - 공식 Capabilities 예시: 카메라, 스크린샷, 브라우저 제어 등
  - 정의되지 않은 기능은 `system_run` 등 별도 경로로 허용 리스트(allow list)를 열어줘야 함
- 노드 등록 후 리모트 게이트웨이가 자동으로 노드를 인식해 해당 노드의 기능 목록 노출

### 4. 다중 디바이스 제어 데모 [03:07:48]
- 구성: OCI(리모트 게이트웨이) + 맥 스튜디오 + 라즈베리파이5 + 윈도우 PC(WSL2)
  - 맥: 사파리, 라즈베리파이: Chromium, 윈도우: 크롬 브라우저 사용
  - 윈도우에서는 네이티브 설치보다 **WSL2** 환경 권장 (네이티브는 불안정)
- 리모트 게이트웨이에서 에이전트가 세 디바이스의 브라우저를 동시에 다른 URL로 전환하는 시나리오 시연
  - 각 노드로 명령이 내려가는 통로와 응답 속도가 달라 약간의 시간 차 발생
- 브라우저 제어가 가능하면 사실상 모든 UI 조작 가능

### 5. 컴패니언 앱 UI 및 게이트웨이 전환 [03:11:37]
- 컴패니언 앱 대시보드에서 리모트 게이트웨이, 연결된 노드 목록, Capabilities 확인 가능
- OpenClaw 3.12 업데이트로 UI 대폭 개선 → 업그레이드 권장
- 드롭다운으로 로컬/리모트 게이트웨이 전환 가능하며, 실제로는 둘 다 동시에 운영 가능
  - 로컬과 리모트가 직접 통신하지는 않지만, 공유 문서를 매개로 협업 구성도 가능할 것으로 예상

### 6. 보안 및 권한 구조 [03:17:17]
- 승인 체계가 여러 레이어로 구성됨: 디바이스 승인 → 노드 승인 → 기능(Capability) 승인
- `system_run`/`exec` 등 비정의 기능을 허용하려면 리모트 게이트웨이에서 allow list를 수동으로 열어줘야 해 번거로울 수 있음

## 핵심 포인트 및 세부 내용

- **최소 요구 사양**: OpenClaw 공식 가이드 기준 라즈베리파이 4 (4GB) 이상, 그 이하 모델은 설치 불가
- **Tailscale 활용**: 리모트 노드 간 네트워크 복잡도를 해소하는 가장 간단한 방법으로 강력 추천
- **에이전트는 리모트 게이트웨이에만 존재**: 각 디바이스의 노드는 단순한 기능 제공자(펑션 서버) 역할, 자체 에이전트 없음
- **멀티 에이전트 확장 가능성**: 로컬 게이트웨이와 리모트 게이트웨이를 공유 문서 등을 통해 간접 연동하면 기종을 넘나드는 멀티 에이전트 구성 가능
- **모니터링 활용 사례**: 여러 디바이스를 한 에이전트가 감시하며 장애 발생 시 자동 복구 명령 전달 가능

---

## 결론 및 권장사항

OpenClaw의 리모트 게이트웨이와 노드 구조를 이해하면, OCI 무료 인스턴스를 허브로 삼아 이기종 디바이스(macOS, Linux, Windows)를 단일 에이전트로 통합 제어할 수 있다. Tailscale로 네트워크를 구성하고 각 디바이스에 OpenClaw 노드를 설치하는 것이 가장 빠른 시작 방법이며, 브라우저 자동화부터 시스템 모니터링·자동 복구까지 다양한 IoT 및 자동화 시나리오에 응용할 수 있다.
