# 정우석 - OpenClaw 스킬 효율적으로 붙이기

## 개요

비개발자인 발표자가 OpenClaw(오픈 클로)를 세 개의 인스턴스로 나눠 운영하면서 스킬을 효율적으로 개발하고 자동화 파이프라인을 구축한 경험을 공유한다. 요청 → 실행 → 고정 → 트리거의 4단계 프로세스를 통해 스킬을 개발하고, OpenClaw와 Claude Code의 역할을 분담하여 콘텐츠 제작 자동화를 구현한 사례를 소개한다.

---

## 주요 주제/섹션

### 자기소개 및 세 가지 OpenClaw 운영 환경 [03:43:46]
- IT 플래닝 팀 소속 비개발자로서 새로운 도구에 대한 관심이 많아 발표 신청
- M백 PC, 맥북, 시놀로지 서버에서 각각 다른 용도로 OpenClaw를 운영
  - 메인(PC): 가능한 스킬을 모두 설치해 범용 목적으로 활용
  - 맥북: 이메일/캘린더 연동 스킬 자동 개발, 교육 문의 초안·커리큘럼 제안 자동화
  - 시놀로지: Vault에 자료를 모아 Nextcloud로 그룹별 공유, Bash 전체 권한 부여

### OpenClaw 스킬 개발 4단계 프로세스 [03:46:14]
- 1단계 **요청**: URL과 함께 "뭐뭐 할 수 있어? 어떤 방식으로 실행돼? 어떤 우려가 있어?" 질문
- 2단계 **실행**: 계획을 먼저 수립하고 반복 피드백 후 실행(계획 없이 실행하면 방향이 엇나가는 경우 발생)
- 3단계 **고정**: "지금 실행한 걸 스킬로 고정하자"고 지시하면 스킬로 자동 저장
- 4단계 **트리거**: 트리거 워드 및 문서화(다큐멘테이션)를 요청해 스킬 완성

### AI 블로그 자동화 시스템 구축 [03:47:20]
- 2월 중순부터 Obsidian + Quartz 4를 활용한 AI 블로그 운영 시작
- 시놀로지 Vault의 자료를 심링크로 Quartz에 연결, 퍼블리싱 자동화
- OpenClaw가 관심 채널(메이커, 에반 등)을 4시간마다 모니터링하여 신규 콘텐츠 요약·발행
- 목표: 1년 후 하루 1,000명 이상 방문하는 블로그 구축
- 장애 대응: Termux(핸드폰 앱)로 접속해 Claude Code와 OpenClaw를 재기동하는 구조 마련

### 콘텐츠·미디어 제작 자동화 [03:49:42]
- 맥북에 NotebookLM 연동, YouTube Data API로 트렌드 아이템 캐치 후 콘텐츠 아이디어 저장
- OpenClaw 단독 운영의 한계: 결과물 검수 부재로 품질이 아쉬웠고, 스킬 수정에도 품질 개선이 제한적
- Claude Code 도입: 영상 URL을 분석해 프레임 단위 애니메이션을 추출하고 재현하는 스킬 구현
- OpenClaw(기획·트렌드 캐치) + Claude Code(영상 제작·실행)의 역할 분담 파이프라인 구성

### OpenClaw + Claude Code 역할 분담 및 파이프라인 [03:51:57]
- **OpenClaw**: 기획, 오케스트레이션, 검수
- **Claude Code**: 복잡한 작업, 제작·실행, 고도화된 스킬 개발
- **서브 에이전트(세션 툴)**: 워크플로우 절차를 지정하면 서브 에이전트를 자동 생성해 순서대로 실행
- 복잡한 명령은 "네가 Claude Code를 진행하면 좋겠다"고 지시하면 자동으로 세팅
- n8n + MCP 조합도 시도했으나, Claude Code로 넘기거나 자체 스킬을 만드는 편이 더 효율적이라는 결론

## 핵심 포인트 및 세부 내용

- **비개발자도 스킬 개발 가능**: 계획 → 피드백 → 실행 → 고정의 단계를 따르면 코딩 지식 없이도 스킬 생성 가능
- **Google CLI URL 제공만으로 전체 스킬 자동 개발**: 구글 클리 주소를 주면 캘린더, 이메일 등 관련 스킬을 일괄 개발
- **시놀로지 Bash 전체 권한 부여**: 시스템 전반에 접근 가능하도록 설정(보안 리스크 인지하고 의도적으로 선택)
- **모바일 장애 대응**: Termux 앱으로 서버에 접속해 Claude Code와 OpenClaw를 재기동하는 안정성 확보
- **n8n MCP 연동 시도 후 철회**: n8n 워크플로우 자체에 MCP를 붙이는 것보다 Claude Code 스킬 방식이 더 효율적

## 결론 및 권장사항

OpenClaw와 Claude Code는 각자 잘하는 영역이 다르므로, OpenClaw는 기획·오케스트레이션·검수에, Claude Code는 복잡한 제작·실행에 역할을 분담하여 함께 사용하는 것이 가장 효율적이다. 비개발자도 4단계(요청 → 실행 → 고정 → 트리거) 프로세스와 계획 수립 단계를 충실히 따르면 품질 높은 스킬을 개발하고 자동화 파이프라인을 구축할 수 있다.
