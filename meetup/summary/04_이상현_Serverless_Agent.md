# 이상현 (Kurt Lee / Mirror) - Mac Mini 없이 Serverless로 Agent 만들기

## 개요

로컬 머신(Mac Mini 등)에서 에이전트를 운영하는 기존 방식의 문제점(보안 노출, 확장성 부재, 비용 낭비)을 지적하고, 클라우드 네이티브·서버리스 아키텍처로 에이전트를 구축한 실제 구현 사례를 소개한다. TypeScript 코드 생성 기반의 에이전트 루프를 AWS Lambda 위에서 구동하고, 로컬 데스크탑 접근은 MQTT 프로토콜로 안전하게 분리하는 구조를 데모와 함께 시연했다.

---

## 주요 주제/섹션

### 1. 기존 로컬 에이전트(Mac Mini) 아키텍처의 문제점 [03:25:49]
- LM 프로바이더, 에이전트 루프, 이벤트 큐, 코드 샌드박스, 상태 저장소를 모두 단일 로컬 머신에 올리는 구조
- 외부 서비스(텔레그램 등)와 통신하려면 퍼블릭 인터넷에 노출 필요 → 포트 포워딩, 공유기 설정 등 일반 사용자에게 비현실적
- 코드 샌드박스를 내 컴퓨터에서 직접 실행하는 보안 위험
- 백업/확장성(Scalability) 부재: 머신이 죽으면 로그 확인 불가, 복제하려면 Mac Mini를 추가 구매해야 함
- SaaS가 죽었다는 담론에 대한 반론: 일반 사용자가 직접 소프트웨어를 만들어 로컬에서 돌리는 것은 비현실적이며, 클라우드 없는 확장성은 불가능

### 2. 서버리스 클라우드 네이티브 아키텍처 설계 목표 [03:29:42]
- **서버리스**: 에이전트를 사용하지 않는 동안에는 비용 발생 없음
- **스테이트리스(Stateless)**: 상태를 외부 저장소에 두어 수평 확장 용이
- **TypeScript 기반 코드 생성**: LLM이 TypeScript 코드를 작성하고, tsc 컴파일 실패 시 자동 재시도 → 타입 세이프티 보장
- **인스펙터블(Inspectable)**: 로컬 파일/시스템 접근 내역을 모두 확인 가능하게 설계
- **외부 의존성 최소화**: Obsidian, Notion 등 없이 메모리 시스템 직접 구현

### 3. 아키텍처 구조 설명 [03:31:34]
- 경계: 로컬(데스크탑 앱) vs. 클라우드(에이전트 루프, 상태, API)
- 데스크탑 앱 ↔ 클라우드 에이전트 루프: **MQTT** 프로토콜(웹소켓 위 경량 실시간 통신)로 연결
- 외부 서비스(텔레그램 등)는 웹 API만 호출 → 로컬 머신까지 도달하지 않음
- 로컬 머신은 아웃바운드 MQTT 커넥션만 유지, 퍼블릭 IP·포트 포워딩·Tailscale 불필요

### 4. 라이브 데모 [03:32:47]
- 텔레그램 메시지 → 클라우드 에이전트 루프 실행 → TypeScript 코드 생성·실행 → 응답 반환 흐름 시연
- 음성 메시지 처리: 오디오 파일 다운로드 → STT → LLM 판단 → TTS 응답 (스킬 모듈 추가로 구현)
- 웹 채팅 인터페이스: 퍼블릭 도메인, 구글 로그인 지원
- 비용: 테스트 포함 약 1~2달러 수준

### 5. 데스크탑 접근 데모 [03:36:49]
- 클라우드 에이전트에 "데스크탑 스킬" 탑재
- MQTT로 데스크탑 앱에 명령 전송 → bash 커맨드 실행 → 결과 반환
- 예시: 디스크 용량 조회(`df -h`), AppleScript로 Chrome 실행
- 레이턴시는 LLM 응답 대기가 대부분이므로 버지니아 리전도 체감 차이 없음

### 6. Google API 및 MCP에 대한 견해 [03:39:48]
- Google Calendar, Gmail 등은 이미 REST API가 존재하므로 TypeScript 타입 정의만 작성하면 바로 호출 가능
- MCP나 CLI 없이도 API 직접 호출로 충분하다는 관점

## 핵심 포인트 및 세부 내용

- **서버리스의 핵심 이점**: 에이전트 미사용 시 과금 없음. 실제 비용은 메시지 수십 건에 $1~2 수준
- **TypeScript 코드 실행 방식**: LLM이 직접 TypeScript 코드를 작성 → tsc로 타입 체크 → 통과 시 실행. 타입 오류 시 LLM이 재작성. 별도 툴 정의 없이 타입 정의 파일만으로 안전한 도구 호출 구현
- **MQTT 활용**: 로컬 머신이 인바운드 포트를 열지 않고도 클라우드와 실시간 양방향 통신 가능
- **확장성**: 에이전트 루프 인스턴스만 복제하면 되며, 상태는 외부 저장소에 분리되어 있어 즉시 수평 확장 가능
- **Lambda 과금 이슈**: LLM 응답 대기 시간도 Lambda 실행 시간으로 과금됨. Lambda 하이버네이션(분할 실행) 방식으로 해결 가능하나, 비용이 매우 낮아 현재는 미적용
- **개발 기간**: 2025년 2월 12일 시작, 약 한 달 만에 구현 완료

## 결론 및 권장사항

- 개인 에이전트를 로컬 머신(Mac Mini 등)에서 운영하는 방식은 보안, 확장성, 유지보수 측면에서 근본적인 한계가 있으며, 클라우드 네이티브·서버리스 구조가 현실적 대안이다.
- TypeScript 기반 코드 생성 + tsc 타입 체크 루프라는 단순한 설계만으로도 안전하고 확장 가능한 에이전트를 한 달 안에 구축할 수 있으며, 실제 운영 비용도 극히 낮다.
