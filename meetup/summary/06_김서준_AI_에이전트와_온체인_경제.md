# 김서준 (Simon Kim / HASHED) - AI 에이전트와 온체인 경제: 다음 10년의 아키텍처

## 개요

HASHED 대표 김서준이 바이브 코딩의 개인 경험을 소개하며, AI 에이전트가 경제 활동의 주체가 되는 시대에 필요한 세 가지 인프라 기둥(MCP, 에이전트 간 통신 프로토콜, 온체인 결제)을 설명한다. 특히 에이전트 신원 인증 표준인 ERC-8004를 중심으로, 에이전트들이 블록체인 기반으로 신뢰하고 결제하고 일하는 인프라가 형성되고 있음을 논한다.

## 주요 주제/섹션

### 자기 소개 및 바이브 코딩 재입문 경험 [03:55:37]
- 컴퓨터공학 전공 후 개발자로 활동, 이후 창업과 투자로 전환해 15년간 개발에서 손을 뗌
- 2025년 11월 Claude Opus 4.5 출시 이후 충격을 받고 날마다 4~5시간씩 바이브 코딩 재개
- 첫 프로젝트: 온체인 데이터 30개를 입력하자 화장실 다녀온 사이 20개의 차트가 자동 생성됨
  - 이더리움 밸류에이션 대시보드 "Instval"을 4시간 만에 완성, 트위터에서 크립토 임팩트 랭킹 1위 기록
- 아부다비행 비행기 안에서 구글 리뷰 데이터 10만 개를 긁어 여행 최적화 앱을 즉석에서 제작, 착륙 직후 경영진 미팅 아이스브레이킹에 활용

### 바이브 코딩이 바꾼 개발의 본질 [04:00:43]
- 과거: 개발자 비용이 스타트업 펀딩의 주된 이유 (구글도 전체 비용의 55%가 개발자 연봉)
- 현재: "어떻게 만들지"보다 "무엇을 만들어야 하는지"가 핵심
- 깊은 기술 구현력보다 현실 문제를 파악하는 도메인 인사이트와 암묵지를 가진 사람이 중요해짐
- HASHED 내부 목표: 2026년까지 VC 중 가장 에이전틱한 조직, 20명이 1,000명 규모의 업무 수행
  - AI가 피치덱 스크리닝, 경쟁사 비교 분석, 포트폴리오-전략적 투자사 매칭 자동화 추진

### 에이전트 경제의 세 가지 인프라 기둥 [04:06:27]
- **MCP (Model Context Protocol)**: 에이전트의 명함, 어떤 역할을 수행할 수 있는지 표현
- **에이전트 간 통신 프로토콜**: 에이전트끼리 교류하는 공용어, 구글이 주도(A2A 프로토콜)
- **온체인 결제 프로토콜 (X40)**: 에이전트가 은행 계좌 없이 자율적으로 지갑을 보유·거래하는 표준

### 에이전트 신원 인증의 필요성과 ERC-8004 [04:07:49]
- 현재 전 세계 수억 개의 에이전트가 활동하나, 신뢰를 확인할 수단이 전무
- 직원 채용 시 이력서·신분증 확인처럼, 에이전트에게도 KYD(Know Your Agent) 과정 필요
- ERC-8004: 세 개 레이어로 구성된 에이전트 신원 인증 표준
  - **레지스트리**: ERC-721 기반 에이전트 출생 신고 및 정보 등록
  - **레퓨테이션 레지스트리**: 실제 블록체인 트랜잭션이 발생한 경우에만 리뷰 적립 (영수증 기반 평점)
  - **밸리데이션**: 평판 확인(경량 작업) → 슬래싱(자산 담보, 실수 시 손실) → 수학적·하드웨어 증명(완전 검증)
- 참여 주체: MetaMask, Google, Ethereum Foundation, Coinbase 등 어벤져스급 연합
  - 특정 기업이 독점 불가한 블록체인 기반이어야만 전 세계 표준으로 채택 가능

### 에이전트 경제 시나리오: 유럽 여행 예약 [04:16:43]
- 개인 에이전트 → 여행 에이전트 → 이탈리아·파리 현지 에이전트 → 실시간 공실·할인 협상 → 선금 결제
- 이 흐름이 신뢰 없이 작동하려면 평판·결제 정보의 블록체인 기록이 필수

### 한국 에이전트 생태계의 폐쇄성 문제 [04:13:47]
- 한국 대기업·빅테크 중 API를 운영하는 회사가 손에 꼽을 정도
- 특정 플랫폼이 OpenAI 마켓플레이스에는 연결하면서 외부 개발자에게는 API 비공개
- 전략적 파트너십 = 데이터 폐쇄의 다른 이름; 성문을 열고 교역량을 높이는 것이 더 부강한 생태계
- 에이전트 세계에서 한국 서비스 기반 프로덕트를 만드는 팀이 거의 없는 현실

### HASHED의 투자 방향 및 마무리 [04:18:57]
- 에이전트 × 블록체인 교차 영역에서 초기 팀 발굴·투자 진행
- 소수 팀이 글로벌 오픈소스 커뮤니티를 대상으로 성장하는 방식이 Web3와 유사
- 서비스 "오모다 도시": OpenAI를 한국형 MCP(배달의민족, 쿠팡 등)와 연결하는 B2C SaaS 준비 중

## 핵심 포인트 및 세부 내용

- 바이브 코딩은 의사결정 단계와 개발 단계를 동시에 극적으로 단축함
- 구글 전체 비용의 55%가 개발자 연봉 → AI로 이 구조가 근본적으로 변화 중
- B2B·B2C를 넘어 **B2A(Business to Agent)** 비즈니스 모델이 부상할 것
- 에이전트 인프라 = MCP(명함) + A2A 통신 + 온체인 결제, 그리고 빠진 한 축이 **신원 인증(여권)**
- ERC-8004는 아직 스마트 컨트랙트 취약점, 결제 범위 제한, 개발자 경험 미흡 등 한계 존재
- 인간이 수만 년에 걸쳐 구축한 신뢰 아키텍처가 에이전트 세계에서는 18개월 만에 압축 재현됨
- 한국에서 에이전트 신원 인증을 시도하는 팀은 현재 발표자 시야에서 없음

## 결론 및 권장사항

AI 에이전트가 경제 활동의 실질적 주체가 되려면 MCP·통신 프로토콜·온체인 결제에 더해 **신원 인증(ERC-8004류 표준)** 이 필수이며, 이 인프라는 블록체인에서만 중립적으로 구축될 수 있다. 에이전트 관련 서비스를 개발하는 이들은 밸류 캡처를 위해 금융·신뢰 레이어와의 연결 가능성을 함께 고려해야 하며, 특히 한국 생태계의 폐쇄적 API 환경 개선과 에이전트 신원 인증 분야에 유망한 기회가 있다.
