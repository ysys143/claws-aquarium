# Zoe Chen (Unibase) - Extending OpenClaw with Unibase: Plug-in Memory, Agent Interoperability & On-Chain Execution

## 개요
- Unibase의 Zoe Chen이 OpenClaw에 플러그인 형태로 추가할 수 있는 세 가지 스킬(메모리, 에이전트 상호운용성, 온체인 실행)을 소개한다. AI가 ChatGPT와 같은 질의응답 모델에서 자율적으로 작동하는 에이전트로 전환되는 흐름 속에서, 에이전트가 기억을 유지하고 서로 협력하며 블록체인 기반 결제와 거래를 독립적으로 수행할 수 있도록 하는 인프라를 제시한다.

---

## 주요 주제/섹션

### AI의 패러다임 전환: 자율 에이전트의 등장 [06:47:29]
- ChatGPT 등 LLM 기반 질의응답 모델에서 24/7 자율 실행 에이전트로의 전환
- 에이전트가 인간 개입 없이 경제 활동에 참여하는 "머신 네이티브 마켓" 개념 소개
- AI 에이전트 전용 플랫폼 "Cloud News"에서 에이전트들이 스스로 "어떻게 기억하는가?"를 토론하는 사례 언급
- 에이전트의 핵심 과제로 "지속적 기억(Persistent Memory)" 문제 제기

### 스킬 1: 탈중앙화 메모리 레이어 [06:49:33]
- Unibase는 자율 에이전트를 위한 탈중앙화 메모리 레이어로 정의
- 다양한 기기(Mac Mini, PC, 스마트폰 등)에 걸쳐 OpenClaw의 메모리를 마이그레이션 가능
- GitHub 링크를 OpenClaw에 전달하면 메모리 스킬을 자동 설치
- 에이전트가 매일 축적되는 컨텍스트와 지식을 기반으로 자기 진화(self-evolving) 가능
  - 사용자가 별도로 학습을 지시하지 않아도 자율적으로 학습 및 발전

### 스킬 2: 에이전트 상호운용성 프로파일(AIP) [06:52:13]
- AIP(Agent Interoperability Profile): 에이전트 간 협력·조율 프로토콜
- 미래 시나리오: 여행 계획 시 항공사 에이전트, 호텔 에이전트 등이 자동으로 협력하여 예약 처리
- 에이전트 간 협력을 위해 필요한 두 가지 요소:
  - **신원(Identity)**: 어떤 에이전트인지, 어떤 능력을 보유하는지 식별
  - **결제(Payment)**: X402·EIP-4 프로토콜로 에이전트 간 직접 결제(인간 개입 불필요)
- 데모: "포춘 쿠키 에이전트"를 Virtuals ACP에 등록하여 다른 에이전트가 서비스를 발견하고 결제할 수 있도록 구현

### 스킬 3: 온체인 실행 (Beat Agent) [06:55:40]
- 탈중앙화 메모리 레이어와 연계한 온체인 실행 스킬
- 두 가지 핵심 기능:
  - **토큰 런칭**: OpenClaw 에이전트에게 토큰 발행 명령 → Base·Ethereum 생태계에 직접 런칭
  - **자율 트레이딩**: 전략 및 리스크 관리 설정 후 에이전트가 자동으로 암호화폐 거래 수행
- OpenClaw Singapore 창업자도 유사 사례(에이전트로 주식·코인 거래) 언급

---

## 핵심 포인트 및 세부 내용
- Unibase는 "AI 인프라" 프로젝트이며, 메모리 레이어 외에도 Beat Agent(토큰 런칭 플랫폼)가 별도 제품으로 존재
- 각 스킬은 독립적: 메모리가 필요하면 메모리 스킬, 거래가 필요하면 Beat Agent 스킬을 별도 설치
- 설치 방식이 매우 간단: GitHub 링크를 OpenClaw에 전달하면 에이전트가 자동으로 스킬 설치
- 에이전트 간 결제에 X402 및 EIP-4 표준을 활용하여 완전한 자율 경제 구현
- 탈중앙화 메모리는 기기 교체 시 기억 손실 문제를 해결하는 실용적 솔루션

---

## 결론 및 권장사항
- OpenClaw에 Unibase의 세 가지 스킬(메모리, AIP, 온체인 실행)을 플러그인 형태로 추가하면 에이전트가 기억을 유지하고, 다른 에이전트와 자율적으로 협력·결제하며, 블록체인 기반 거래까지 독립적으로 수행할 수 있다. GitHub 링크를 통한 간편한 설치로 OpenClaw의 자율성을 크게 확장할 수 있다.
