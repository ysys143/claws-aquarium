# [발표] 한수관 (AER Labs) - ResearchClaw

## 개요
- 리서처 출신의 발표자가 반도체 및 로보틱스 연구를 자동화하기 위해 Claude 기반 에이전트 시스템 "ResearchClaw"를 개발한 경험을 공유한다. 24시간 자율 실행이라는 Claude의 핵심 장점을 연구 자동화에 적용하고자, 직접 제어 가능한 결정론적 FSM(유한 상태 기계) 기반 하네스를 구축하였다. 시뮬레이션 환경에서는 안정적으로 동작하며, 실제 하드웨어 연동을 위한 추가 개선 작업이 진행 중이다.

---

## 주요 주제/섹션

### 1. 발표 배경 및 동기 [06:05:24]
  - 리서처 출신으로 반도체·로보틱스 연구 자동화의 필요성을 오래전부터 느껴왔음
  - 군 복무 중 일과 외 시간에 연구 자동화를 시도하고 싶었으나 여건이 부족했음
  - OpenClaw(Claude 에이전트) 등장 이후 본격적으로 시도 가능해짐

### 2. 기존 접근의 한계 [06:06:45]
  - 단일 에이전트에 모든 것을 맡기면 컨텍스트가 뒤섞이는 문제 발생
  - 평가(Evaluation) 단계와 실험(Experiment) 단계의 코드가 혼재되는 현상
  - 결정론적 제어가 없으면 연구 흐름을 에이전트가 제대로 따르지 못함

### 3. ResearchClaw 시스템 구조 [06:07:07]
  - **플래닝 단계**: 사람과 상호작용하며 연구 계획을 점진적으로 구체화
    - 반도체 설계 시 요구 스펙(Specification) → 구현 스펙(Implementation Spec) 단계별 명세화
  - **실험 루프(Experiment Loop)**: 코드 작성 후 반드시 실행되도록 강제하는 툴콜 구현
  - **평가(Evaluation) 단계**: 시각화(Visualization) 시스템이 병목 지점으로, 별도 설계 필요

### 4. 하드웨어 연동 및 평가 시스템 [06:08:27]
  - 로보틱스: 뎁스(Depth) 카메라와 FPGA 기반 매니퓰레이션 시스템 구축
  - 실제 보드에 RTL을 올린 후 센싱 데이터를 에이전트에 피드백하는 루프 구성
  - 시뮬레이션 → 실제 환경 전환 시 추가 챌린지(레이턴시, 툴콜 해석 오류 등) 존재

### 5. 결과 및 데모 [06:09:26]
  - 심플한 픽앤플레이스(Pick and Place) 테스트에서 정상 동작 확인
  - 반도체 설계(RTL 코드·디자인) 자동 생성도 가능함을 확인
  - 실제 태스크 영상은 추후 별도 공유 예정

### 6. 오픈소스 공개 [06:10:04]
  - ResearchClaw 프로세스는 GitHub 리포지토리(researchclaw)에 공개 중
  - 연구 자동화에 관심 있는 분들에게 참고 자료로 활용 권장

### 7. Q&A [06:10:32]
  - **질문**: LLM 단독으로 연구 전 과정(주제 선정~실험)을 맡겼을 때 잘 안 됐는데, 어떤 개입 기준을 두는가?
  - **답변**: 결정론적 FSM 루프로 플래닝 / 실행 / 평가 단계를 명확히 분리하고, 각 단계 에이전트의 권한(디렉토리, 역할)을 명시적으로 제한하는 방식으로 해결

## 핵심 포인트 및 세부 내용
- Claude 에이전트의 24시간 자율 실행 능력을 연구 자동화에 적용하는 것이 핵심 아이디어
- 반도체 설계는 탐색 공간(Search Space)이 방대하여 자동화의 효과가 특히 큼
- 단일 에이전트보다 **단계별로 역할을 분리한 결정론적 FSM 구조**가 연구 자동화에 더 적합
- 각 에이전트에게 특정 디렉토리 내에서만 작업하도록 권한을 제한하는 것이 안정성의 핵심
- 시뮬레이션 환경에서는 충분히 잘 동작하나, 실제 하드웨어(FPGA, 로봇) 연동 시 레이턴시·툴콜 해석 오류 등의 디버깅 과제가 남아 있음
- 로보틱스 실험 평가를 위한 시각화 시스템 구축이 현재 가장 큰 병목

---

## 결론 및 권장사항
- 연구 자동화를 위해 에이전트에게 모든 것을 맡기기보다, **플래닝 → 실험 → 평가** 단계를 명확히 나누고 각 단계의 역할과 권한을 제한하는 결정론적 설계가 중요하다.
- ResearchClaw GitHub 리포지토리를 통해 연구 자동화 파이프라인의 구체적인 구현 방식을 참고할 수 있으며, 유사한 연구 자동화를 시도하는 이들에게 실용적인 출발점이 될 수 있다.
